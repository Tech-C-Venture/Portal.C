# Implementation Plan

- [x] 1. CSV共通ユーティリティの実装
- [x] 1.1 CSV生成関数の実装
  - データ配列とカラム定義からUTF-8 BOM付きCSV文字列を生成する関数を作成する
  - ヘッダー行の出力、フィールド値のエスケープ処理（カンマ・ダブルクォート・改行含有時のクォート囲み）を実装する
  - 空データの場合はヘッダー行のみを出力する
  - _Requirements: 5.1, 5.2, 5.4_

- [x] 1.2 CSVパース関数の実装
  - CSV文字列を受け取り、ヘッダー行とデータ行の配列に分解する関数を作成する
  - 先頭のBOM（`\uFEFF`）が存在する場合は自動除去する
  - ダブルクォートで囲まれたフィールドの基本的な解析に対応する
  - _Requirements: 5.1, 5.3, 5.5_

- [x] 1.3 CSVヘッダー検証関数の実装
  - パースされたヘッダー行が期待するカラム定義と一致するかを検証する関数を作成する
  - 不一致の場合は期待するカラム名を含むエラーメッセージを返す
  - _Requirements: 7.2_

- [x] 2. 共通時間割CSVダウンロードAPIの実装
- [x] 2.1 (P) 共通時間割のCSVダウンロードエンドポイントを作成する
  - 管理者認証チェックを行い、非管理者には403を返す
  - Firestoreから共通時間割データ（`is_public=true`）を取得する
  - カラム「学年, 専攻, 曜日, 時限, 教科名, 教室, 担当講師」でCSVを生成する
  - 曜日はDB上の数値を日本語表記（月, 火, 水...）に変換して出力する
  - レスポンスにContent-Dispositionヘッダーを設定し、ファイル名に当日日付を含める（例: `timetables_2026-04-10.csv`）
  - データ0件の場合はヘッダー行のみのCSVを返す
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 6.1, 6.2_

- [x] 3. 時間帯マスターCSVダウンロードAPIの実装
- [x] 3.1 (P) 時間帯マスターのCSVダウンロードエンドポイントを作成する
  - 管理者認証チェックを行い、非管理者には403を返す
  - Firestoreから全時間帯マスターデータを取得する
  - カラム「時限, ラベル, 開始時刻, 終了時刻, 有効」でCSVを生成する
  - 「有効」カラムはboolean値を`true`/`false`文字列として出力する
  - ファイル名に当日日付を含める（例: `time_slots_2026-04-10.csv`）
  - データ0件の場合はヘッダー行のみのCSVを返す
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 6.1, 6.2_

- [x] 4. 共通時間割CSVアップロードServer Actionの実装
- [x] 4.1 共通時間割CSVの行バリデーション処理を実装する
  - 各行について学年（1-4）、曜日（日本語→数値変換）、時限（1-10）、教科名（空でないこと）、専攻（有効な専攻名であること）を検証する
  - エラーがある場合は行番号（ヘッダー除く1始まり）とエラー内容を記録する
  - 全行を検証し、1行でもエラーがあれば全行分のエラーリストを返す
  - _Requirements: 3.3, 3.4, 3.6_

- [x] 4.2 共通時間割の一括上書きServer Actionを実装する
  - FormDataからCSVファイルを取得し、テキストとしてパースする
  - ファイル形式チェック（CSV以外はエラー）とヘッダー検証を行う
  - バリデーション通過後、Firestoreバッチで既存の共通時間割（`is_public=true`）を全削除し、CSVの内容で新規登録する
  - 時間帯マスターから時限番号に対応する`time_slot_id`を逆引きして設定する
  - 成功時は登録件数を返し、該当ページをrevalidateする
  - サーバーエラー発生時はバッチをコミットせず、エラーメッセージを返す
  - _Requirements: 3.1, 3.4, 3.5, 5.5, 6.1, 6.2, 7.1, 7.5_

- [x] 5. 時間帯マスターCSVアップロードServer Actionの実装
- [x] 5.1 時間帯マスターCSVの行バリデーション処理を実装する
  - 各行について時限（1-10）、ラベル（空でないこと）、開始時刻（HH:MM形式）、終了時刻（HH:MM形式）、開始<終了チェック、有効フラグ（true/false/1/0）を検証する
  - エラーがある場合は行番号とエラー内容を記録する
  - _Requirements: 4.3, 4.4, 4.6_

- [x] 5.2 時間帯マスターの一括上書きServer Actionを実装する
  - FormDataからCSVファイルを取得し、パース・ヘッダー検証・行バリデーションを行う
  - バリデーション通過後、Firestoreバッチで既存の時間帯マスターを全削除し、CSVの内容で新規登録する
  - 成功時は登録件数を返し、該当ページをrevalidateする
  - _Requirements: 4.1, 4.2, 4.5, 5.5, 6.1, 6.2, 7.1, 7.5_

- [x] 6. CSVアップロード/ダウンロードUIと管理者ページ統合
- [x] 6.1 共通時間割のCSVアップロードフォームコンポーネントを作成する
  - CSVファイル選択（`.csv`のみ受付）とアップロードボタンを配置する
  - アップロードボタンクリック時に確認ダイアログを表示し、既存データが上書きされる旨を警告する
  - アップロード中はローディング状態を表示し、ボタンの二重クリックを防止する
  - 成功時は成功メッセージと登録件数を表示する
  - エラー時はエラー行番号とエラー内容の一覧を表示する
  - _Requirements: 3.1, 3.2, 3.5, 7.1, 7.3, 7.4_

- [x] 6.2 (P) 時間帯マスターのCSVアップロードフォームコンポーネントを作成する
  - 共通時間割アップロードフォームと同様の構造で、時間帯マスター用Server Actionにバインドする
  - 確認ダイアログ、ローディング状態、エラー/成功表示を実装する
  - _Requirements: 4.1, 4.2, 4.5, 7.1, 7.3, 7.4_

- [x] 6.3 管理者時間割ページにCSVダウンロードボタンとアップロードフォームを統合する
  - 共通時間割セクションにCSVダウンロードリンクとアップロードフォームを追加する
  - 時間帯マスターセクションにCSVダウンロードリンクとアップロードフォームを追加する
  - 既存の手動フォーム入力機能を維持したまま、CSV操作UIを隣接配置する
  - _Requirements: 6.3_
