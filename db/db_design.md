#　DB設計案

##　必要なもの

### 1.Member（メンバー）
「学年」は保存せず、「入学年度」から計算する形に集約します。

- ID (ZITADELの識別子)
- 名前
- 学校メール (公開)
- Gmailアドレス (非公開・管理者用)
- 入学年度 (学年計算用)
- 留年フラグ (学年計算調整用)
- 所属専攻
- 現在のステータス (テキスト)
- ステータス更新日時 (24時間判定用)
- 管理者フラグ (権限管理用)

### 2.Tag (タグ)
独立した項目として管理し、検索しやすくします。

- タグ名 (例: Python, 競技プログラミング)
- 紐付け(どのメンバーがどのタグを持っているか)


### 3.Timetables (時間割)
「どの専攻・学年のための時間割か」というマスターデータと、「誰がそれを履修しているか」を管理します。

- 授業名（例：プログラミング基礎、線形代数）
- 曜日・時限（例：月曜・2限）
- 対象専攻（例：情報工学）
- 対象学年（例：1年生 ※入学年度と計算して合致するか判定）
- 教室・場所
- 紐付け（どのメンバーがこの授業を登録しているかのリスト）


### 4. Events (イベント)
- イベントID (一意識別用)
- タイトル・説明文
- 開催日時・場所
- 定員
- 作成者ID (誰が作ったか)
- 参加者リスト (誰が参加しているか)
